HTML support 2019-06-03

This post is written in HTML! This may not be very obvious yet, because the default style puts everything in monospace, and a wall of monospaced text looks pretty similar regardless of whether it comes from a HTML or TXT source file.

One obvious difference is that HTML files ignore carriage-return (and blank lines in general) so if you actually want to start a new paragraph, you will need to use either the p or br tag.

And here is some monospaced text in a "pre" block.  It doesn't look terribly different than the text above, because everything is monospace anyway, and long lines are configured to wrap even in a "pre".  One difference is that carriage returns ARE respected in a "pre", so things like ASCII art will work.  For best results, you will want to keep the art to less than 60 columns wide (or whatever width is specified in the default CSS file).
